# Code of Conduct

## Our standard

Texere is committed to a respectful, technically serious, and inclusive collaboration environment.

Examples of behavior that help create that environment include:

- being respectful in technical disagreement
- giving actionable, specific feedback
- assuming good intent while still being clear about problems
- prioritizing the health of the project and its contributors over ego or point-scoring
- welcoming contributors with different backgrounds and levels of experience

Examples of unacceptable behavior include:

- harassment, discrimination, or intimidation
- personal attacks or hostile language
- deliberately disruptive behavior in issues, pull requests, or discussions
- publishing private information about someone without permission
- bad-faith technical review intended to embarrass or block others

## Scope

This Code of Conduct applies within project spaces, including issues, pull requests, discussions,
review comments, and other project-associated communication channels.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, pull requests, or other
contributions that violate this Code of Conduct.

Repeated or severe violations may result in temporary or permanent exclusion from project
participation.

## Reporting

If you experience or witness unacceptable behavior, report it privately to the maintainers through
GitHub rather than escalating publicly.

For security-sensitive reports, use the process in [`SECURITY.md`](./SECURITY.md) instead.
