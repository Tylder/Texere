Idea: The system should have the ability to self-inspect so that the user should be able to ask
questions about the system, for example: "how do I do X", "Can you do?", "Where in the system is Y
defined?", etc. It must therefore be possible to select the system it self as scope.

I also see it possible for one of the first features this system adds is to it self.

I also want to be able to ask questiona about "how" a feature or project what worked on, meaning a
graph of work. I can see parallel ideation leading to problems, leading to request, leading to
specs, leading more research, leading to going back to ideation, and eventaully implementation. The
purpose of this is to hopefully improve how we work and use this system. it might also highlight
some ways to improve the system.

I could even imagine using a separate orchestration system with access to the knowledge graph and
code that simulates being a human user and works with the system to implement a feature or project.
The purpose for this would again be that the user orchestrator could find ways to improve both the
system and the user experience, it could for example notice information that would be useful to the
user but isnt shown or perhaps ways to work with the user that allows you to help the user and give
them the information they need for them to come to the best conclusion. Its basically like wrapping
everything in a critique loop.
