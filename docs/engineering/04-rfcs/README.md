# 04-RFCs (Proposals / Discussions)

**Purpose:** Explore options and trade-offs before committing.

RFCs are where thinking and debate happens. They exist to prevent costly-to-reverse choices from
being made without explicit exploration of alternatives.

## File naming

```
RFC-<area>-<topic>.md
```

Examples:

- `RFC-orchestration-resumability-mechanism.md`
- `RFC-provider-registry-storage-strategy.md`

## Rules

- Disposable after decisions are made
- Must link to resulting ADRs
- Must link to PROBDEF drivers
- Status lifecycle: **Draft** → **In review** → **Accepted** → **Closed** (or **Rejected**)

## Minimum required content

- Problem statement (must link to PROBDEF problems)
- Goals and non-goals
- Constraints
- Options considered (at least 2 where realistic)
- Trade-offs (explicit)
- Recommendation
- Open questions

## What belongs here

- Constraints
- Options considered
- Trade-offs
- Recommendation
- Open questions

## What does NOT belong here

- Binding obligations
- Final decisions (those go in ADRs)
