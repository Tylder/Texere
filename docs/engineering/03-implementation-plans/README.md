# 03-Implementation Plans

Defines the execution roadmap: milestones, work breakdown, risks, and rollout strategy.

Implementation Plans are **operational** documents that coordinate delivery. They evolve as
execution reveals sequencing changes.

**Key:** One Implementation Plan can coordinate multiple Specifications and Requirements.

## How to contribute

When Specifications are ready to implement, create one Implementation Plan:

- `IMPL-PLAN-<area>-<topic>.md` – Milestones, work breakdown, risks, verification

See `../_templates/IMPL-PLAN-template.md` for template.

After creating a document:

- [ ] Update this README (add to Active section)
- [ ] Update `../DOCUMENT-REGISTRY.md` (add entry)
- [ ] Link to all Specs and Requirements this plan covers

---

## Naming Convention

- Feature: `IMPL-PLAN-export-feature.md`
- Cross-cutting: `IMPL-PLAN-pagination-system.md`
- Infrastructure: `IMPL-PLAN-performance-optimization.md`

---

## Tips

- Link only upward (to Specs and Requirements, not Ideation)
- Include multiple Specs if they're coordinated by one plan
- Verify milestones prove conformance to involved Specs
- Include risk register and mitigation strategies
- Track actual vs. planned completion for retrospectives
