# Reference Documents

Design drivers, analyses, frameworks, and knowledge references that inform decisions across all
phases but don't fit the phase-based categories (Ideation → Requirements → Specifications →
Implementation Plans).

Use these when you need to understand:

- System constraints or design drivers that shape what gets built
- Analyses of gaps, problems, or tool landscapes
- Knowledge frameworks or agent reasoning guides
- Foundational concepts and terminology

## Active Reference Documents

- [`REFERENCE-agent-knowledge-requirements`](REFERENCE-agent-knowledge-requirements.md) – Questions
  that effective coding agents must be able to answer when modifying a repository

## How to contribute

Add a document here when it:

- Provides foundational knowledge or constraints used across multiple phases
- Analyzes problems, gaps, or design space without directly proposing features
- Defines frameworks or reasoning guides for agents or developers
- Should be referenced by multiple Ideation, Requirements, or Spec documents

Naming convention:

- `ANALYSIS-<topic>.md` – Gap analyses, landscape studies
- `FRAMEWORK-<topic>.md` – Reasoning guides, question sets
- `CONCEPTS-<topic>.md` – Terminology, definitions, mental models
- Or use descriptive names that fit the content
