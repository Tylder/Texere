# .indexer-config.json Guide

This is a template for the Texere Indexer configuration file. Copy this file to
`.indexer-config.json` in your project root and customize as needed.

## File Location & Precedence

1. **Runtime** (CLI flags) — highest priority

   ```bash
   pnpm indexer run --config ./path/to/config.json
   ```

2. **Per-repo** (optional for third-party repos) `./.indexer-config.json` at repository root

3. **Global** (orchestrator configuration) File pointed to by `INDEXER_CONFIG_PATH` env var or
   `./.indexer-config.json` at app root

4. **Defaults** — hardcoded fallbacks Used when no config file is found and system allows missing
   config

**See:**
[configuration_and_server_setup.md §8](./docs/specs/feature/indexer/configuration_and_server_setup.md#8-precedence-hierarchy)
for details.

## Required vs Optional

### Required Fields

- `version` — Config file schema version (currently `"1.0"`)
- `codebases[].id` — Unique identifier per codebase
- `codebases[].root` — Absolute path to repository root
- `codebases[].trackedBranches` — Git branches to index (≥1 required)
- `graph.neo4jUri`, `.neo4jUser`, `.neo4jPassword` — Neo4j credentials
- `vectors.qdrantUrl` — Qdrant instance URL

### Optional Fields (All Have Sensible Defaults)

- `security`, `embedding`, `llm`, `worker` — See tables in
  [config-reference.md](./docs/guides/config-reference.md#optional-fields-with-defaults)

## Environment Variable Substitution

Fields like `neo4jUri` support `${VAR_NAME}` syntax for secrets:

```json
"neo4jUri": "${NEO4J_URI}",
"neo4jPassword": "${NEO4J_PASSWORD}"
```

Populate in shell or `.env` file (never commit `.env`):

```bash
# .env (not committed)
export NEO4J_URI=bolt://localhost:7687
export NEO4J_USER=neo4j
export NEO4J_PASSWORD=your-password
export QDRANT_URL=http://localhost:6333
export OPENAI_API_KEY=sk-...
```

Then load before running indexer:

```bash
source .env
pnpm indexer run
```

---

## Validation

Always validate configuration before indexing:

```bash
pnpm indexer validate --config ./.indexer-config.json
```

Validation checks:

- JSON syntax validity
- All required fields present
- Field types and constraints
- Environment variables resolved
- Database connections reachable

## Examples

### Local Development (Docker)

```json
{
  "version": "1.0",
  "codebases": [
    {
      "id": "my-repo",
      "root": "/home/user/my-repo",
      "trackedBranches": ["main"]
    }
  ],
  "graph": {
    "neo4jUri": "bolt://localhost:7687",
    "neo4jUser": "neo4j",
    "neo4jPassword": "neo4j"
  },
  "vectors": {
    "qdrantUrl": "http://localhost:6333"
  }
}
```

### Production (With Env Vars)

```json
{
  "version": "1.0",
  "codebases": [
    {
      "id": "production",
      "root": "/opt/repos/production",
      "trackedBranches": ["main"]
    }
  ],
  "graph": {
    "neo4jUri": "${NEO4J_URI}",
    "neo4jUser": "${NEO4J_USER}",
    "neo4jPassword": "${NEO4J_PASSWORD}"
  },
  "vectors": {
    "qdrantUrl": "${QDRANT_URL}"
  },
  "worker": {
    "type": "bullmq",
    "concurrency": 16
  }
}
```

## IDE Integration (VSCode)

For autocomplete and validation in `.vscode/settings.json`:

```json
{
  "json.schemas": [
    {
      "fileMatch": [".indexer-config.json"],
      "url": "./node_modules/@repo/indexer-core/dist/schemas/indexer-config.schema.json",
      "description": "Texere Indexer configuration schema"
    }
  ]
}
```
